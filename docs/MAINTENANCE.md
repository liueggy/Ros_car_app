# Repository layout

This repository contains the native mobile clients for Eggy ROS Car.

```text
.
├── android/
│   └── EggyRosCar/          # Android Gradle project
├── ios/
│   └── EggyRobotClient/     # SwiftUI iOS project generated by XcodeGen
├── docs/                    # Maintenance notes
└── .github/workflows/       # CI build and Release publishing
```

## Release strategy

The mobile clients publish into one shared rolling Release:

- Tag: `mobile-latest`
- Name: `Eggy ROS Car Mobile Latest`
- Assets:
  - `EggyRosCar-debug.apk`
  - `ROS-Car-unsigned.ipa`

Android and iOS workflows are intentionally independent. Rebuilding one platform updates its own asset in the shared Release without deleting the other platform's asset.

Do not use separate rolling tags such as `android-latest` or `unsigned-latest` for the default downloadable builds; they make the GitHub Releases page look like one platform replaced the other.

## Local development

### iOS

```bash
cd ios/EggyRobotClient
brew install xcodegen
xcodegen generate
open EggyRobotClient.xcodeproj
```

### Android

```bash
cd android/EggyRosCar
gradle assembleDebug
```

## CI workflows

- `.github/workflows/ios-client.yml`
  - builds unsigned device IPA on push to `ios/**`
  - uploads the IPA as a workflow artifact
  - publishes `ROS Car-unsigned.ipa` to `mobile-latest`
  - also builds a simulator app artifact for debugging
- `.github/workflows/android-client.yml`
  - builds debug APK on push to `android/EggyRosCar/**`
  - uploads the APK as a workflow artifact
  - publishes `EggyRosCar-debug.apk` to `mobile-latest`

## Signing notes

The default iOS Release asset is unsigned. Sign it locally with a trusted sideload signer or configure the manual signed IPA workflow secrets if you need an ad-hoc signed IPA:

- `IOS_CERTIFICATE_P12`
- `IOS_CERTIFICATE_PASSWORD`
- `IOS_PROVISIONING_PROFILE`
